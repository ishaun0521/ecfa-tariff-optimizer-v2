稅則判定流程教學

版本 v0.5.0 更新時間：2026-03-12（台灣時間） 改版歷程

這一頁是給使用者、同事與客戶看的教學頁，目的是解釋：台灣出口到中國大陸的商品，稅則是怎麼判的，以及 ECFA 優惠為什麼不能跳過稅則歸類直接談。

重點：本平台有研究與前置判斷能力，但不冒充中國海關最終裁定。最終稅則與優惠適用，仍以中國海關接受申報、預裁定與正式審核為準。

一張圖看懂流程

這張圖可以直接拿來做說明：先判商品事實與候選號列，再看 ECFA 貨品清單，再看原產地與文件，最後才談優惠是否可能落地。

核心觀念：不是先問有沒有 ECFA，而是先問商品到底是什麼

  • 第一層：稅則歸類——先判商品本質、成分、用途、製程、包裝型態。
  • 第二層：ECFA 貨品清單——看這個號列是否在協議覆蓋範圍內。
  • 第三層：原產地認定——看是否符合完全取得 / 實質轉型 / 區域價值含量等規則。
  • 第四層：文件要求——確認能否用證明書、製程、BOM、發票等資料支撐前面三件事。

台灣出口端在做什麼？

  1. 整理商品名稱、成分、用途、製程、包裝與交易條件
  2. 找出候選稅則號列
  3. 檢查是否可能落在 ECFA 貨品清單
  4. 準備原產地與 supporting documents
  5. 把一致的資料包交給中國進口商 / 報關行

中國進口海關端在做什麼？

  1. 接受或不接受申報號列
  2. 接受或不接受 ECFA 優惠主張
  3. 審查原產地證明與資料一致性
  4. 必要時要求補件、改列他號、補稅或不予優惠
  5. 有爭議時，以預裁定 / 正式審核結果為準

平台是怎麼判的？

  1. 建立商品事實檔：商品描述、成分/BOM、用途、製程、包裝、目的地。
  2. 做 pre-classification：輸出候選 HS code，而不是假裝只有唯一答案。
  3. 檢查 ECFA 覆蓋：確認候選號列是否在 ECFA 貨品清單內。
  4. 檢查原產地：看台灣來源、製程、附加價值與高風險原料。
  5. 檢查文件：看支撐證據夠不夠、彼此是否一致。
  6. 再做最佳化：如果有空間，才去做 BOM 調整建議。

為什麼我們強調「前置判斷」而不是「海關裁定」？

  • 因為最終進口關稅與優惠適用，真正關鍵是中國海關最後接受的申報號列。
  • 同一個商品名稱，如果成分、包裝或申報型態不同，號列可能完全不同。
  • ECFA 清單與原產地規則都建立在號列之上，不能倒推。
  • 所以平台要做的是：縮小不確定性、提高前期判斷品質、降低溝通成本。

你可以怎麼拿這頁去介紹別人？

  • 先給對方看流程圖，讓他理解「稅則 → ECFA 清單 → 原產地 → 文件」的順序。
  • 再進首頁示範輸入商品與 BOM，讓他看到平台如何輸出候選號列與風險提示。
  • 最後補一句邊界聲明：這是 pre-classification，不是假裝替海關下裁定。
